replication.io is a community center for databases and services that support Apache CouchDB-style replication. We are working to expand the definition and specification to encompass the growing family of databases that support HTTP-based, multi-master replication.

Objective

Apache CouchDB was created with multi-master replication built into its foundation. The list of databases supporting this multi-master replication style has grown over the past half decade.

The goal of this project is to combine our communities around our shared strength.

Beyond Databases?

The replication format and process is a simple exchange of document and revision identifiers. It has been implemented in single node, multi node, mobile, and even web browser-based databases. It can go even farther. Anything that can support a by-sequence index and tracks changes to its content (be it a database, a file system, web service, or perhaps even hardware and sensors), could support multi-master replication.

Initial work has been done to define the originating implementation done in Apache CouchDB (see Specification). Work is underway to provide a specification as well as testing environments that encompasses this family of databases, and provides a framework for future implementors to use in testing their databases, filesystems, services, or hardware.

Collaborate and Contribute

We encourage anyone with an interest and/or implementation in multi-master replication to join this documentation and testing effort.
